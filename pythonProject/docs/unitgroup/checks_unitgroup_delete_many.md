# Таблица проверок PATCH /UnitGroup/DeleteMany

| № | Название проверки | Что проверяется | Ожидаемый результат |
|---|-------------------|-----------------|---------------------|
| 1 | Успешное удаление двух групп — 200 и схема | Создать 2 группы, PATCH /UnitGroup/DeleteMany с массивом из двух id | 200, deleted/count/results; схема RestResponseDto; обе группы отсутствуют в GET |
| 2 | Запрос без авторизации — 401 | PATCH DeleteMany без Basic Auth | 401 Unauthorized |
| 3 | Пустой массив [] — 200, deleted=0 | Тело [] | 200, deleted=0 или отсутствует |
| 4 | Тело не массив — 400 | Один объект вместо массива | 400, 415 или 422 |
| 5 | Массив с несуществующими id — 200, частичный результат | Тело [ { id: 999999999 }, { id: 999999998 } ] | 200 или 400; при 200 — в ответе results или deleted/count |

---

## Подробные тест-кейсы

### ТК-1. Успешное удаление двух групп — 200 и схема

**Предусловия:** Созданы две UnitGroup через PUT; получены их id из GET /UnitGroup.

**Шаги:**
1. Выполнить PATCH /UnitGroup/DeleteMany с Basic Auth и телом [ { id: id1 }, { id: id2 } ].
2. Проверить код 200.
3. Проверить наличие deleted/count/results и что удаление зафиксировано (deleted >= 1 или count >= 1 или results не пусты).
4. Проверить ответ по схеме RestResponseDto.
5. Выполнить GET /UnitGroup и убедиться, что групп с id1 и id2 нет в items.

**Ожидаемый результат:** 200; ответ по схеме RestResponseDto; обе группы удалены и отсутствуют в GET.

---

### ТК-2. Запрос без авторизации — 401

**Шаги:**
1. Выполнить PATCH /UnitGroup/DeleteMany с телом [ { id: 1 } ] без заголовка Authorization.

**Ожидаемый результат:** HTTP 401 Unauthorized.

---

### ТК-3. Пустой массив [] — 200, deleted=0

**Шаги:**
1. Выполнить PATCH /UnitGroup/DeleteMany с телом [] и Basic Auth.
2. Проверить код 200 и что deleted=0 или поле отсутствует/допустимо.

**Ожидаемый результат:** HTTP 200; deleted=0 или эквивалент.

---

### ТК-4. Тело не массив — 400

**Шаги:**
1. Отправить PATCH /UnitGroup/DeleteMany с телом { id: 1 } (один объект вместо массива) и Basic Auth.

**Ожидаемый результат:** HTTP 400, 415 или 422.

---

### ТК-5. Массив с несуществующими id — 200, частичный результат

**Шаги:**
1. Отправить PATCH /UnitGroup/DeleteMany с телом [ { id: 999999999 }, { id: 999999998 } ] и Basic Auth.
2. Допустимый ответ: 200 с results/deleted/count (частичный результат) или 400.

**Ожидаемый результат:** 200 с структурой ответа (results или deleted/count) либо 400.
