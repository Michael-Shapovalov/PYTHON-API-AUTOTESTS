# Проверки PATCH /Unit/DeleteMany (удаление массива объектов)

| № | Название проверки | Что проверяется | Ожидаемый результат |
|---|-------------------|-----------------|---------------------|
| 1 | Успешное удаление двух единиц — 200 и схема | Создать 2 единицы, DeleteMany с массивом из двух id | 200, deleted, схема RestResponseDto, обе единицы отсутствуют в GET |
| 2 | Запрос без авторизации — 401 | PATCH без auth | 401 |
| 3 | Пустой массив [] — 200, deleted=0 | Тело [] | 200, без падения, deleted=0 или отсутствует |
| 4 | Тело не массив — 400 | Один объект вместо массива | 400, 415 или 422 |
| 5 | Массив с одним несуществующим id | Один валидный + один несуществующий id | 200, results или deleted в ответе |

---

## Подробные тест-кейсы

### ТК-1. Успешное удаление двух единиц — 200 и схема

**Предусловия:** Созданы две единицы через PUT; получены их id из GET /Unit.

**Шаги:**
1. Выполнить PATCH /Unit/DeleteMany с Basic Auth и телом [ { id: id1 }, { id: id2 } ].
2. Проверить код 200.
3. Проверить наличие deleted/count/results и что удаление зафиксировано (deleted >= 1 или count >= 1 или results не пусты).
4. Проверить ответ по схеме RestResponseDto.
5. Выполнить GET /Unit и убедиться, что единиц с id1 и id2 нет в items.

**Ожидаемый результат:** 200; ответ по схеме RestResponseDto; обе единицы удалены.  
**Примечание:** На момент анализа тест падает на валидации схемы (та же причина, что и для Delete по id): в results[].entity приходят поля id, integrationId, code, name, схема их не разрешает. См. FAILED_TESTS_ANALYSIS.md.

---

### ТК-2. Запрос без авторизации — 401

**Шаги:** PATCH /Unit/DeleteMany с телом [ { id: 1 } ] без Authorization.

**Ожидаемый результат:** 401 Unauthorized.

---

### ТК-3. Пустой массив [] — 200, deleted=0

**Шаги:**
1. Выполнить PATCH /Unit/DeleteMany с телом [] и Basic Auth.
2. Проверить код 200 и что deleted=0 или поле отсутствует/допустимо.

**Ожидаемый результат:** HTTP 200; deleted=0 или эквивалент.

---

### ТК-4. Тело не массив — 400

**Шаги:** Отправить PATCH /Unit/DeleteMany с телом { id: 1 } (один объект вместо массива) и Basic Auth.

**Ожидаемый результат:** HTTP 400, 415 или 422.

---

### ТК-5. Массив с одним несуществующим id

**Шаги:**
1. Отправить PATCH /Unit/DeleteMany с телом [ { id: 999999999 }, { id: 999999998 } ] и Basic Auth.
2. Допустимый ответ: 200 с results/deleted/count (частичный результат) или 400.

**Ожидаемый результат:** 200 с структурой ответа (results или deleted/count) либо 400.
