# Проверки POST /Unit (получение объектов по фильтру)

| № | Название проверки | Что проверяется | Ожидаемый результат |
|---|-------------------|-----------------|---------------------|
| 1 | Успешный ответ 200 и структура ответа | Код 200, наличие полей пагинации и items; созданная единица в items | 200, поля pageNum, totalPages, pageSize, total, hasPreviousPage, hasNextPage, items; единица по integrationId в items |
| 2 | Соответствие ответа схеме PaginatedListOfRestUnit | Валидация ответа по jsonschema из swagger | Ответ проходит jsonschema.validate |
| 3 | Запрос без авторизации — 401 | POST без auth | 401 |
| 4 | Пустой body — 400 | POST с телом {} | 400 (или 200 по контракту) |
| 5 | Невалидный JSON в теле — 400/415 | POST с обрезанным JSON | 400, 415 или 422 |
| 6 | ids не массив — 400 | ids передано строкой вместо массива | 400 или ошибка в ответе |

---

## Подробные тест-кейсы

### ТК-1. Успешный ответ 200 и структура ответа

**Предусловия:** Создана одна единица через PUT (фикстура unit_created_for_post).

**Шаги:**
1. Сформировать тело запроса: pageNum=0, pageSize=100, integrationIds=[integrationId созданной единицы], noCount=false.
2. Выполнить POST /Unit с Basic Auth и этим телом.
3. Проверить код ответа 200.
4. Проверить наличие в ответе полей: pageNum, totalPages, pageSize, total, hasPreviousPage, hasNextPage, items.
5. Убедиться, что items — массив и в нём есть единица с указанным integrationId.

**Ожидаемый результат:** HTTP 200; структура ответа соответствует пагинации; созданная единица присутствует в items.

---

### ТК-2. Соответствие ответа схеме PaginatedListOfRestUnit

**Предусловия:** Есть созданная единица (фикстура).

**Шаги:**
1. Выполнить POST /Unit с телом { pageNum: 0, pageSize: 0 } и Basic Auth.
2. Загрузить схему PaginatedListOfRestUnit из swagger.
3. Выполнить jsonschema.validate(instance=response.json(), schema=schema).

**Ожидаемый результат:** HTTP 200; валидация схемы проходит без ошибки.

---

### ТК-3. Запрос без авторизации — 401

**Шаги:**
1. Выполнить POST /Unit с телом { pageNum: 0, pageSize: 0 } без заголовка Authorization.

**Ожидаемый результат:** HTTP 401 Unauthorized.

---

### ТК-4. Пустой body — 400

**Шаги:**
1. Выполнить POST /Unit с Basic Auth и телом {}.

**Ожидаемый результат:** HTTP 400 или HTTP 200 (если API допускает пустой body).

---

### ТК-5. Невалидный JSON в теле — 400/415

**Шаги:**
1. Отправить POST /Unit с Basic Auth и телом в виде невалидного JSON (например, обрезанная строка `{"pageNum": 0, "pageSize": `).

**Ожидаемый результат:** HTTP 400, 415 или 422.

---

### ТК-6. ids не массив — 400

**Шаги:**
1. Отправить POST /Unit с телом { pageNum: 0, pageSize: 0, ids: "not-an-array" } и Basic Auth.

**Ожидаемый результат:** HTTP 400 или иная ошибка валидации; либо 200 с валидной структурой ответа, если API игнорирует поле.
