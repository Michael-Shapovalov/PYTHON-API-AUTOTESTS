# Проверки PATCH /Unit/Delete (удаление одного объекта)

| № | Название проверки | Что проверяется | Ожидаемый результат |
|---|-------------------|-----------------|---------------------|
| 1 | Успешное удаление по id — 200 и схема | Создать единицу, удалить по id, проверить 200, deleted, схему, отсутствие в GET | 200, deleted=1, ответ по схеме RestResponseDto, единицы нет в GET |
| 2 | Успешное удаление по integrationId — 200 | Создать единицу, удалить по integrationId | 200, deleted или count |
| 3 | Запрос без авторизации — 401 | PATCH без auth | 401 |
| 4 | Удаление несуществующего id | id, которого нет в БД | 200 с deleted=0 или 404 |
| 5 | Пустой body — 400/404 | Тело {} | 400, 404 или 422 |
| 6 | Невалидный JSON — 400 | Обрезанный JSON | 400, 415 или 422 |

---

## Подробные тест-кейсы

### ТК-1. Успешное удаление по id — 200 и схема

**Предусловия:** Создана одна единица через PUT; получен id единицы из GET /Unit.

**Шаги:**
1. Выполнить PATCH /Unit/Delete с Basic Auth и телом { id: id_единицы }.
2. Проверить код 200, наличие deleted/count/results, соответствие ответа схеме RestResponseDto.
3. Выполнить GET /Unit — единицы с этим id нет в items.

**Ожидаемый результат:** 200; ответ по схеме RestResponseDto; единица удалена.  
**Примечание:** На момент анализа тест падает на валидации схемы: в results[].entity API возвращает поля id, integrationId, code, name, а схема BaseDto в swagger не разрешает свойства (additionalProperties: false без properties). См. FAILED_TESTS_ANALYSIS.md.

---

### ТК-2. Успешное удаление по integrationId — 200

**Шаги:**
1. Создать единицу с integrationId. PATCH /Unit/Delete с телом { integrationId } и Basic Auth.
2. Проверить 200 и deleted или count/results.

**Ожидаемый результат:** 200; удаление зафиксировано.

---

### ТК-3. Запрос без авторизации — 401

**Шаги:** PATCH /Unit/Delete с телом { id: 999999 } без Authorization.

**Ожидаемый результат:** 401 Unauthorized.

---

### ТК-4. Удаление несуществующего id

**Шаги:** PATCH /Unit/Delete с телом { id: 999999999 } и Basic Auth.

**Ожидаемый результат:** 200 (deleted=0) или 404 или 400.

---

### ТК-5. Пустой body — 400/404

**Шаги:** PATCH /Unit/Delete с телом {} и Basic Auth.

**Ожидаемый результат:** 400, 404 или 422.

---

### ТК-6. Невалидный JSON — 400

**Шаги:** PATCH /Unit/Delete с телом в виде невалидного JSON.

**Ожидаемый результат:** 400, 415 или 422.
